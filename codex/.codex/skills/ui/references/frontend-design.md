# Frontend Design Summary

提升 UI 独特性和美学完成度的设计指南。
仅在明确要求时应用。

## 设计思考（首先确定）
- Purpose: 解决谁的什么问题的 UI
- Tone: 选择一个大胆的方向（例: minimal / brutalist / editorial / playful）
- Constraints: 技术约束和 a11y
- Differentiation: 确定该 UI 的"令人难忘的一点"

## 实现质量标准
- Production-grade 实际可运行
- 视觉印象强烈，方向一致
- 细节打磨（间距、层级、状态）

## 设计指南
- Typography: 引人注目的 display 和易读的 body 字体组合
- Color: 明确主色和锐利的强调色
- Motion: 集中于一次大型演出（不要过于分散）
- Space: 有意选择留白或高密度
- Background: 有氛围感的图层或质感（如需要）

## 注意
- 方向的关键是"意图的强度"，而非是否华丽。
- 实现复杂度应与设计方向相匹配。
