# NotebookLM Slide Design YAML (2 Variations)

## 目的

在 NotebookLM 的"幻灯片资料"生成中，**通过 YAML 明确设计方向来提高品质和可复现性**。

此技能为即使相同来源也能改变外观，输出**意图理解方式不同的两个方案**的 YAML。

参考（使用方法/示例）:
- `https://note.com/yoshifujidesign/n/nd9c8db0b55b8`
- `https://note.com/yoshifujidesign/n/n7412bccb5762`

---

## 使用方法（NotebookLM 侧）

在 NotebookLM 中加载资料来源后，
［幻灯片资料］→［自定义幻灯片资料］→［请描述要创建的幻灯片］
中**粘贴此技能输出的 YAML 全文**后［生成］。

---

## 输入（向用户确认的事项）

如果可能，最初最多确认以下5个问题，如无回答则用括号中的默认值继续。

1. 目的（例: 内部分享/销售/招聘/面向投资者）（内部分享）
2. 预期读者（例: 经营层/开发/非工程师）（非工程师混合）
3. 基调（例: 信赖感/创新/亲切）（信赖感）
4. 品牌元素（logo/品牌色/字体指定）（无指定）
5. 照片/图表比例（照片较多/数据较多）（数据较多）

---

## 输出（必需格式）

必须输出以下 **2个方案**。

- 方案A: **可读性·信赖感·数据传达最优先**（简约/企业风格）
- 方案B: **重视情感·故事·留白美学**（编辑/生活风格）

两者都用**中文 YAML**，形式上可以直接粘贴到 NotebookLM。

输出格式:

- `### 方案A` 正下方放一个 YAML（代码块）
- `### 方案B` 正下方放一个 YAML（代码块）

---

## 生成规则

- 具体颜色要包含 **HEX**（例：`#FFFFFF`）
- 也要写一些"不要做的事"（例：禁止过多框线、禁止过度阴影、限制颜色数量）
- 章节结构/导航/留白/网格等，**明确化影响可复现性的约束**
- 即使内容（来源）不明也要成立，写成**结构模板**

---

## 输出模板（生成示例骨架）

### 方案A（简约/企业风格）

（根据需要，按目的/读者/品牌色微调）

```yaml
# presentation_design_spec_minimal_corporate_cn.yaml
# 风格: 现代简约 / 企业风格
# 目的: 最优先可读性和数据传达（信赖感）

全局设计设置:
  基调: "信赖感、专业、简洁、理性"
  配色方案:
    基础: "#FFFFFF (白色) 或 #F5F5F7 (极浅灰)"
    文字色: "#111111 (近乎黑) 或 #333333 (炭灰)"
    强调色: "如有品牌色则使用。否则采用 #0052CC (蓝色)"
    高亮: "#FFB020 (琥珀色) - 仅限重要数值和注意事项"
    颜色限制: "最多3色 + 灰色阶"
  排版:
    标题: "较粗的黑体（无衬线）。简短有力的词语。"
    正文: "可读性高的黑体。行间距较大。"
    数字: "无衬线粗体放大。单位（%等）稍小。"
  通用布局规则:
    网格: "12列对齐。左右留白较宽。"
    导航: "左上角小字显示 '01. SECTION' 等编号+章节标题。"
    留白: "禁止堆砌。一页幻灯片一个信息。"
    图表: "表格线/网格线最少。仅强调点用强调色。"
  禁止事项:
    - "禁止过度投影"
    - "禁止过多装饰框"
    - "禁止过多渐变"

布局变体（目录）:
  - 类型: "封面"
    设计: "留白大。标题左对齐放大，副标题小字。一条浅横线。"
  - 类型: "章节扉页"
    设计: "章节标题放大。背景单色简约。"
  - 类型: "结论（TL;DR）"
    设计: "结论3点 + 一个大数字/指标。"
  - 类型: "双栏（问题 vs 解决）"
    设计: "竖线分割。左:问题，右:解决。标题粗体，正文细体。"
  - 类型: "数据/图表"
    设计: "柱状/折线简洁表现。注释简短。"
  - 类型: "流程"
    设计: "水平步骤（01-05）。圆形图标最小化。"
  - 类型: "时间线"
    设计: "竖线 + 年份大 + 说明小。"
  - 类型: "总结"
    设计: "下一步行动3点。清单式简洁。"
```

### 方案B（编辑/故事风格）

（根据需要，按目的/读者/照片比例微调）

```yaml
# presentation_design_spec_editorial_story_cn.yaml
# 风格: 现代编辑 / 生活杂志风
# 目的: 用留白和照片讲述故事（情感/说服）

全局设计设置:
  基调: "温和、知性、情感、有机、精致"
  视觉识别:
    背景色: "#F3F0EB (沙米色) 或 #EBEBEB (暖灰)"
    文字色: "#333333 (炭灰) - 避免纯黑"
    强调色: "#E07A5F (赤陶色) 或 #708D81 (橄榄绿)"
    图片风格:
      特征: "自然光、胶片颗粒、饱和度较低"
      形状: "圆角矩形 或 拱形"
  排版:
    标题: "优雅的宋体/衬线体。重要词适度强调。"
    正文: "易读的黑体/无衬线，稍小。"
    数字: "衬线数字增添沉稳。"
  通用布局规则:
    留白: "留白最优先。降低文字密度。"
    照片: "基本每页一张大照片。"
    分隔线: "禁止框线。分隔仅用留白或极细线。"
  设计规则:
    - "禁止冷色·霓虹色（统一暖色调）"
    - "禁止过多图标（用照片和排版讲述）"

布局变体（目录）:
  - 类型: "封面（杂志封面）"
    设计: "右侧照片全出血，左侧标题。细横线分隔。"
  - 类型: "引用（Quote）"
    设计: "短消息居中。背景淡显大引号。"
  - 类型: "故事（1→2→3）"
    设计: "竖向步骤。点线连接。每步旁边小圆形照片。"
  - 类型: "比较"
    设计: "左右两张照片 + 各自说明。颜色克制。"
  - 类型: "数据"
    设计: "极细折线 + 说明段落补充（附上故事）。"
  - 类型: "总结"
    设计: "下一步行动3点。留白收尾。"
```
