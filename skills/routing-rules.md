# Skill Routing Rules (SSOT)

スキル間のルーティングルール（トリガー/除外）を一元管理。
各スキルはこのファイルを参照し、自身の `description` には概要のみ記載すること。

> **重要**: 各スキルファイルにローカルなトリガー/除外テーブルを作成しないこと。
> 変更が必要な場合は、このファイルのみを編集する。

## ルーティング判定フロー

```
ユーザー入力
    │
    ├── "Codex" を含む？
    │       │
    │       ├── 除外キーワードにマッチ → 該当スキルを除外
    │       ├── トリガーキーワードにマッチ → 該当スキルをロード
    │       └── どちらでもない → デフォルト: codex-review（レビュー目的と推定）
    │
    └── "Codex" を含まない → 通常のスキルマッチング
```

## 優先順位ルール（決定的）

キーワードが複数のスキルにマッチする場合の優先順位:

1. **除外が最優先**: 除外キーワードにマッチしたスキルは絶対にロードしない
2. **具体的なキーワードが優先**: "Codex Worker" > "Codex"
3. **デフォルトスキル**: トリガー/除外どちらにもマッチしない場合は `codex-review` をロード

> **注**: 「文脈判定」は曖昧さを生むため使用しない。すべてのケースは上記ルールで決定的に判定される。

## 更新ルール

1. **SSOT 原則**: ルーティングルールの変更はこのファイルのみで行う
2. **相互参照**: 各スキルはこのファイルへのリンクを持つ（ローカルテーブル禁止）
3. **description 簡潔化**: スキルの description には概要のみ、詳細はここを参照
4. **完全リスト維持**: 汎用表現（"〜全般"）を使わず、具体的なキーワードを列挙する
