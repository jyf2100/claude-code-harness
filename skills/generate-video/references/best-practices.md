# SaaS Video Best Practices

SaaS 介绍视频的最佳实践集。
提供根据视频目的和漏斗选择最佳构成的指南。

---

## 基本原则

### 1. 先于画面决定「消除谁的什么痛点」

不是功能介绍，而是设计成最短创造观众痛点→解决的确信的装置。

**风险**：目标模糊则无法打动任何人

### 2. 最初几秒放弃多余仪式

开头长时间放 logo 和演出会导致流失率上升，因此立即进入正题。
重视声音和正题开始的速度。

### 3. 长度由目的决定

一般视频越长参与度越低，但最佳长度取决于目的。

| 长度 | 用途 |
|------|------|
| 1-2 分钟 | 开头流失少 |
| 5-10 分钟 | 详细解说 |

**结论**：长视频开头设计是关键

### 4. CTA 不要放在最后

为防止观众中途流失，CTA 也要放在中间。

### 5. 易看的视频是卫生管理

**优先级**：音质 > 画面可读性 > 节奏 > 外观

**重要**：声音不好会立即流失

### 6. 字幕和转录是必需的

字幕是必需要求，不仅要自动生成还要修正。

### 7. 长视频需要章节

视频观众想跳着看，所以要加章节（目录）。

## 漏斗别视频类型比较

| 目的（漏斗） | 视频类型 | 长度参考 | 构成核心 | 主要 KPI |
|-------------|---------|---------|---------|---------|
| 认知~兴趣 | 超短 teaser | 30~90 秒 | 痛点→结果（未来）→"后续在这里" | 观看维持/CTR |
| 兴趣→考虑 | Intro（概要演示） | 2~3 分钟 | 完整 1 个用例 | 演示申请/试用开始 |
| 考虑→确信 | 演示（短销售） | 2~5 分钟 | 先消除反对意见 | 商谈转化率/回复率 |
| 确信→决策 | Walkthrough / Webinar | 5~30 分钟 | 实际运用流程+证据 | CVR/咨询 |
| 扩展·效率化 | 混合演示 | 录制+Q&A | 录制标准化→现场个性化 | 成约率/工时削减 |
| 继续·活用 | Onboarding | 30 秒~数分钟（分小段） | 最短快速成功→Aha | Activation/继续率 |
| 支持 | How-to/故障 | 1~5 分钟 | 1 个视频 1 个目的 | 咨询削减/自我解决率 |

## 类别别指南

### LP/广告用：短介绍（30~90 秒）

**目的**：认知~兴趣漏斗

**构成大纲**：
- 0-5 秒：痛点 or 期望结果
- 5-20 秒：目标用户和承诺
- 20-60 秒：象征性工作流 1 个
- 60-90 秒：下一步行动

**应避免的陷阱**：功能罗列·抽象词汇连发

### 考虑用：2~3 分钟 Intro 演示

完整演示 1 个用例，避免专业术语，边分解边说明。

**推荐事项**：
- 开头简短进入正题
- 选择最能打动人的 1 个案例
- 中间也设置 CTA

### 销售用：2~5 分钟演示

推动购买决策下一步行动的短销售视频。

**策略**：
- 先消除前 3 个反对意见
- 1 个视频=1 个行业/职业

### 决策用：15~30 分钟 walkthrough

促进更深入理解和决策的长视频。章节设计必需。

**关键点**：
- 基本流程
- 反对处理
- 管理/安全说明

### Onboarding 用

设计让用户到达第一次成功体验（Aha）。

**指南**：
- 1 个视频不要教全部
- 按角色分叉视频

### 支持/帮助视频

不要让视频成为唯一答案，与文本并用。

**必需元素**：
- 视频+步骤文本
- 字幕+全文转录

## 模板

### 90 秒打动人的介绍模板

**用途**：LP/广告、认知~兴趣漏斗

| 时间 | 内容 | 帧数(30fps) |
|------|------|------------|
| 0:00-0:05 | 痛点 or 期望结果 | 0-150 |
| 0:05-0:15 | 目标用户和承诺 | 150-450 |
| 0:15-0:55 | 象征性工作流 | 450-1650 |
| 0:55-1:10 | 差异化依据 | 1650-2100 |
| 1:10-1:30 | CTA | 2100-2700 |

**合计**：90 秒 = 2700 帧

### 3 分钟 Intro 演示模板

**用途**：考虑用、兴趣→考虑漏斗

| 时间 | 内容 |
|------|------|
| 0:00-0:10 | 结论+痛点 |
| 0:10-0:30 | 用例声明 |
| 0:30-2:20 | 实际画面完整演示 |
| 2:20-2:50 | 打消 1 个常见顾虑 |
| 2:50-3:00 | CTA |

### 20 分钟决策用 walkthrough

**用途**：确信→决策漏斗

| 时间 | 内容 |
|------|------|
| 0:00-1:00 | 对象和课题 |
| 1:00-8:00 | 基本流程 |
| 8:00-12:00 | 前 2 个反对意见 |
| 12:00-15:00 | 管理/安全 |
| 15:00-20:00 | 成功案例+CTA |

## 制作检查清单

### 录制前

- [ ] 台本：精简要说的话
- [ ] 演示环境：关闭通知、去除个人信息
- [ ] 画面：UI 文字可读的缩放

### 录制中

- [ ] 声音：清晰度最优先
- [ ] 灯光/画面：最低限
- [ ] 节奏：操作犹豫的瞬间全部剪掉

### 发布时

- [ ] 字幕：必须质量修正
- [ ] 转录：用于搜索和浏览
- [ ] 长视频必须加章节

## 通用失败模式

| 失败 | 影响 |
|------|------|
| 目标不明确 | 无法打动任何人 |
| Logo·公司介绍太长 | 开头流失 |
| 功能全包含 | 焦点模糊 |
| 无字幕/字幕差 | 可访问性降低 |
| 只放视频就满足 | 无法测量效果 |
| CTA 只在最后 | 中途流失看不到 |

## 推荐 3 个套装

最快出成果的基本套装：

1. **90 秒 teaser** - 获取认知
2. **3 分钟 Intro 演示** - 促进考虑
3. **15~25 分钟决策用 walkthrough** - 支持成约

**成果**：填补获取→考虑→决策的信息空白

## Harness 中的应用

### 视频类型自动判定

| Harness 检测条件 | 推荐视频类型 | 模板 |
|-----------------|-------------|------|
| 新项目 | LP/广告用短视频 | 90 秒 teaser |
| 检测到 UI 更改 | Intro 演示 | 3 分钟模板 |
| CHANGELOG 更新 | 发布说明 | 重视 Before/After |
| 大型结构更改 | 架构解说 | walkthrough |

### 场景构成指南

#### 短视频（30-90 秒）时

```
HookScene (3 秒) → ProblemPromise (7 秒) → WorkflowDemo (40-60 秒) → Differentiator (10 秒) → CTA (10 秒)
```

#### Intro 演示（2-3 分钟）时

```
Hook (10 秒) → UseCase 声明 (20 秒) → 实际画面演示 (110 秒) → 消除顾虑 (30 秒) → CTA (10 秒)
```

#### Walkthrough（15-30 分钟）时

```
对象和课题 (1 分) → 基本流程 (7 分) → 反对处理 (4 分) → 管理/安全 (3 分) → 成功案例+CTA (5 分)
```

## Remotion 实现规则

### 动画

| 规则 | 理由 |
|------|------|
| 必须用 `useCurrentFrame()` | 禁止 CSS animations，使用 Remotion 的帧控制 |
| `spring({ damping: 200 })` | 流畅的动作 |
| `interpolate()` + `extrapolateRight: 'clamp'` | 防止值失控 |
| 打字机效果用 `text.slice(0, charCount)` | 不推荐按字符改 opacity |

### 音频

| 规则 | 理由 |
|------|------|
| `Audio` 从 `@remotion/media` import | 不推荐 `Html5Audio` |
| 音频开始 = 场景开始 + 30f（1 秒等待） | 幻灯片更改后视觉稳定再开始音频 |
| 场景长度 = 30f + 音频长 + 20f 余量 | 过渡前音频结束 |
| 用 `ffprobe` 事先确认音频长度 | 场景设计的前提信息 |

### TransitionSeries

| 规则 | 理由 |
|------|------|
| 过渡长度推荐 15f（0.5 秒） | 自然切换 |
| 场景开始 = 前场景开始 + 前场景长 - 过渡长 | 考虑重叠 |
| 音频在过渡开始前结束 | 切换中的音频会违和 |

### 场景长度计算例

```
音频长度确认（30fps 换算）：
  hook: 4.0 秒 = 121 帧
  problem: 12.0 秒 = 360 帧

场景长度计算：
  hook: 30(等待) + 121(音频) + 24(余量) = 175 帧
  problem: 30(等待) + 360(音频) + 25(余量) = 415 帧

场景开始帧（过渡 15f）：
  hook: 0
  problem: 175 - 15 = 160

音频开始时机：
  hook: 0 + 30 = 30
  problem: 160 + 30 = 190
```
