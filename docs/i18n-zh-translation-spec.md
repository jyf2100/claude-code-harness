## 2. 不翻译术语（Keep as-is）

以下术语在所有上下文中保留英文：

### 2.1 技术名词

- Git, GitHub, GitLab
- CLI, API, SDK, HTTP, HTTPS, JSON, YAML
- npm, yarn, pnpm, bun
- TypeScript, JavaScript, Python, Go, Rust
- Node.js, Deno
- Docker, Kubernetes
- Linux, macOS, Windows
- CPU, GPU, RAM, SSD

### 2.2 命令和代码

- 所有命令名称（如 `harness-work`, `breezing`, `codex exec`）
- 所有文件路径
- 所有变量名、函数名、类名
- 所有配置键名（如 `permissions.defaultMode`）
- 所有环境变量名（如 `CLAUDE_PLUGIN_ROOT`）

### 2.3 状态标记

- `cc:TODO`, `cc:WIP`, `cc:完了`
- `APPROVE`, `REQUEST_CHANGES`
- `pass`, `fail`, `skip`
- `enabled`, `disabled`

### 2.4 产品名称

- Claude Code, Claude
- Codex
- Harness（作为产品名时）
- OpenCode, Cursor

## 3. 翻译风格指南

### 3.1 语气

- **正式但友好**：使用"请"、"可以"、"建议"等礼貌用语
- **简洁明确**：避免冗长表述，直接说明要点
- **主动语态**：优先使用主动语态

**示例：**
```
❌ ファイルの削除を行うことができませんでした
✓ 无法删除文件

❌ 以下のコマンドを実行することをお勧めします
✓ 建议执行以下命令
```

### 3.2 格式

- **消息前缀**：使用中文标签
  - `[错误]` → `[错误]`
  - `[警告]` → `[警告]`
  - `[信息]` → `[信息]`
  - `[成功]` → `[成功]`

- **变量占位**：保持 `{variable}` 或 `$variable` 格式
  ```
  日文: ファイル {filename} が見つかりません
  中文: 文件 {filename} 未找到
  ```

- **列表**：使用 `•` 或 `-` 作为项目符号

### 3.3 标点符号

- 中文文本使用全角标点：`，。：；""''`
- 技术内容中的标点保持原样
- 日文 `、` → 中文 `，`
- 日文 `。` → 中文 `。`
- 日文 `：` → 中文 `：`（中文冒号）

## 4. 上下文敏感翻译

某些术语需要根据上下文选择不同翻译：

### 4.1 "設定"（Settings/Configuration）

| 上下文 | 翻译 |
|--------|------|
| 作为名词（配置项） | 设置 |
| 作为动词（配置操作） | 配置 |
| 文件名 | 配置文件 |

### 4.2 "実行"（Execute/Run）

| 上下文 | 翻译 |
|--------|------|
| 命令执行 | 执行 |
| 脚本运行 | 运行 |
| 测试运行 | 运行 |

### 4.3 "作成"（Create/Generate）

| 上下文 | 翻译 |
|--------|------|
| 文件创建 | 创建 |
| 计划制定 | 制定 |
| 报告生成 | 生成 |

## 5. 脚本输出消息规范

### 5.1 错误消息格式

```bash
# 日文
echo "[エラー] ファイル $file が見つかりません"

# 中文
echo "[错误] 文件 $file 未找到"
```

### 5.2 警告消息格式

```bash
# 日文
echo "[警告] この操作は元に戻せません"

# 中文
echo "[警告] 此操作无法撤销"
```

### 5.3 成功消息格式

```bash
# 日文
echo "[成功] $count 個のファイルを処理しました"

# 中文
echo "[成功] 已处理 $count 个文件"
```

### 5.4 进度消息格式

```bash
# 日文
echo "処理中... ($current/$total)"

# 中文
echo "处理中... ($current/$total)"
```

## 6. 特殊处理规则

### 6.1 双语保留

对于关键说明，可保留日文原文作为注释：

```markdown
<!-- 日文原文: テストは必須です -->
测试是必需的
```

### 6.2 代码注释

代码注释翻译为中文，但保留技术术语：

```bash
# 日文
# ファイルの存在を確認
if [ ! -f "$file" ]; then

# 中文
# 检查文件是否存在
if [ ! -f "$file" ]; then
```

### 6.3 文档标题

SKILL.md 的主标题可翻译，但保持简洁：

```markdown
# 日文
# Harness Work (v3)

# 中文
# Harness Work (v3)  # 保留，因为是技能名
```
