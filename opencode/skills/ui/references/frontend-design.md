# Frontend Design Summary

提高 UI 独特性和美观完成度的设计指南。
仅在明确要求时适用。

## 设计思考（首先决定）
- Purpose: 为谁解决什么问题的 UI
- Tone: 选择 1 个大胆的方向（如: minimal / brutalist / editorial / playful）
- Constraints: 技术约束和 a11y
- Differentiation: 决定该 UI 的「令人印象深刻的 1 点」

## 实现的质量标准
- Production-grade 且实际运行
- 视觉上印象深刻，方向一致
- 细节打磨（间距、层级、状态）

## 设计指南
- Typography: 引人注目的 display 和易读的 body 配对
- Color: 明确主角色和尖锐强调色
- Motion: 集中于 1 次大演出（不过度分散）
- Space: 有意选择留白 or 高密度
- Background: 有氛围的图层或质感（如需要）

## 注意
- 方向重要的是「意图的强度」。不是花哨与否。
- 实现复杂度要配合设计方向。
