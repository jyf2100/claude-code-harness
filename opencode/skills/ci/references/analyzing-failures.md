# CI Analyze Failures

分析 CI/CD 管道失败并确定原因的技能。
解读 GitHub Actions、GitLab CI 等的日志。

---

## 输入

- **CI 日志**: 失败任务的日志
- **run_id**: CI 执行的标识符（如有）
- **仓库上下文**: CI 配置文件

---

## 输出

- **失败原因的确定**: 具体原因
- **修复建议**: 处理方法的建议

---

## 执行步骤

### Step 1: 确认 CI 状态

```bash
# GitHub Actions 的情况
gh run list --limit 5

# 确认最新失败
gh run view --log-failed
```

### Step 2: 获取失败日志

```bash
# 特定 run 的日志
gh run view {{run_id}} --log

# 仅失败步骤
gh run view {{run_id}} --log-failed
```

### Step 3: 分析错误模式

#### 构建错误

```
模式: "error TS\d+:" 或 "Build failed"
可能原因:
- TypeScript 类型错误
- 依赖缺失
- 语法错误
```

#### 测试错误

```
模式: "FAIL" 或 "✕" 或 "AssertionError"
可能原因:
- 测试失败
- 测试超时
- Mock 不匹配
```

#### 依赖错误

```
模式: "npm ERR!" 或 "Could not resolve"
可能原因:
- package.json 不一致
- 私有包认证问题
- 版本冲突
```

#### 环境错误

```
模式: "not found" 或 "undefined"
可能原因:
- 环境变量未设置
- Secret 缺失
- 路径问题
```

## 错误模式字典

### TypeScript 错误

| 错误代码 | 含义 | 典型修复 |
|-------------|------|-------------|
| TS2304 | 找不到名称 | 添加 import |
| TS2322 | 类型不匹配 | 修正类型 |
| TS2345 | 参数类型错误 | 修正参数 |
| TS7006 | 隐式 any | 添加类型注解 |

### npm 错误

| 错误 | 含义 | 典型修复 |
|--------|------|-------------|
| ERESOLVE | 依赖解析失败 | 删除 package-lock & 重装 |
| ENOENT | 找不到文件 | 确认路径 |
| EACCES | 权限错误 | 确认 CI 配置 |

### Jest/Vitest 错误

| 错误 | 含义 | 典型修复 |
|--------|------|-------------|
| Timeout | 测试超时 | 延长超时或修正异步 |
| Snapshot | 快照不匹配 | `npm test -- -u` |

---

## 多个错误的优先级

1. **构建错误**: 最优先修复
2. **依赖错误**: 需要在构建前解决
3. **测试错误**: 构建成功后处理
4. **Lint 错误**: 最后处理

## 注意事项

- **日志很大**: 提取重要部分
- **注意连锁错误**: 找到第一个错误
- **环境差异**: 考虑本地和 CI 的区别
